# Two-Factor Authentication (2FA): The Complete Technical Guide for Security Professionals

## Introduction

Two-Factor Authentication (2FA) has evolved from a recommended security practice to a critical control in modern cybersecurity architectures. This comprehensive guide merges technical protocols, real-world attack vectors, and enterprise implementation strategies into a single authoritative resource for security engineers and network administrators.

## 1. Core 2FA Methods and Cryptographic Foundations

### 1.1 Time-Based One-Time Password (TOTP) - RFC 6238

<div class="code-block">
<pre>
<code class="language-python">
# TOTP Generation Pseudocode
def generate_totp(secret_key):
    time_counter = int(time.time()) // 30
    msg = struct.pack(">Q", time_counter)
    hmac_hash = hmac.new(secret_key, msg, hashlib.sha1).digest()
    offset = hmac_hash[-1] & 0x0F
    binary = (hmac_hash[offset] & 0x7F) << 24
    binary |= (hmac_hash[offset+1] & 0xFF) << 16
    binary |= (hmac_hash[offset+2] & 0xFF) << 8
    binary |= hmac_hash[offset+3] & 0xFF
    return binary % 10**6
</code>
</pre>
</div>

**Technical Implementation Details:**
- Shared secret `K` (160-bit minimum) provisioned via QR code or manual entry
- HMAC-SHA-1 computation every 30 seconds (±1 interval tolerance)
- Server-side validation includes clock drift compensation

**Security Considerations:**
- Phishing susceptibility through real-time proxy attacks
- Seed database compromise enables full TOTP replication
- NIST recommends migration to HMAC-SHA-256 for new implementations
